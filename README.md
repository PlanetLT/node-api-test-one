# Node API Test One

A Node.js/Express API with PostgreSQL (Prisma), Redis caching, rate limiting, and basic auth/watchlist/movie routes. This README focuses on how to run and verify behavior in development.

**Features**
- Express 5 API with structured logging
- PostgreSQL via Prisma
- Redis cache middleware with `X-Cache` header (`HIT` or `MISS`)
- API rate limiting (optionally backed by Redis)
- File uploads and static serving for `/uploads`
- i18n middleware (server-side message translation)

**Requirements**
- Node.js 18+
- PostgreSQL
- Redis

**Quick Start**
1. Install dependencies.
2. Create a `.env` file.
3. Start the server.

```bash
npm install
npm run dev
```

**Environment Variables**
Create `.env` in the project root. Set only what you need.

```ini
# App
PORT=5001
NODE_ENV=development

# Database
DATABASE_URL=postgresql://user:password@localhost:5432/db_name?schema=public

# Redis
REDIS_URL=redis://localhost:6379
REDIS_CACHE_TTL_SECONDS=60
REDIS_CONNECT_TIMEOUT_MS=1000

# Auth (JWT)
JWT_SECRET=change_me
ACCESS_TOKEN_SECRET=change_me
REFRESH_TOKEN_SECRET=change_me
ACCESS_TOKEN_EXPIRES_IN=15m
REFRESH_TOKEN_EXPIRES_IN=7d
ACCESS_TOKEN_COOKIE_MAX_AGE_MS=900000
REFRESH_TOKEN_COOKIE_MAX_AGE_MS=604800000

# Logging (optional)
LOG_LEVEL=info
LOG_TO_FILES=true
PINO_PRETTY=true
```

**Scripts**
- `npm run dev` start with nodemon
- `npm start` start without nodemon
- `npm test` run tests
- `npm run test:watch` watch tests
- `npm run seed:movies` seed movie data

**Cache Behavior**
Routes using the cache middleware return an `X-Cache` header.
- `X-Cache: HIT` response served from Redis
- `X-Cache: MISS` response generated by the route handler

Example:
```bash
curl -i http://localhost:5001/movies
```

**Rate Limiting**
Default policy is 100 requests per 15 minutes per IP. If Redis is available, rate limit state is stored in Redis.
